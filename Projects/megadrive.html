Mega Drive Project

Making a Mega Drive game in 2021

The SEGA Mega Drive is quite an old console right? But that's no reason for not trying to make a game, so here I am, procrastinating while trying to accomplish my goal to complete a full game that can runs on original hardware without any problem.

The reason

Sometimes while I'm really bored I start thinking about weird projects to do and suddenly a random video pops up and for some reason that was about making a game for the NES.

When I went to search how expensive that was I remembered that I had a Mega Drive at home so after a quick Google search I found the solution with the name of SGDK.

After some time of doing nothing I got something, a ball moving across the screen, right after that I wanted more so I decided to work a bit on a project and then start a YouTube channel with the main goal of keeping me motivated for finishing that amazing project.

How

Nowadays, we make games using engines like Unity or Unreal, but there is still a few people that want to make games for "retro" consoles, such as the NES or in this case, the Mega Drive. This type of games receive the name of Homebrew Games, for being games made by hobbyists for proprietary hardware.

But what enables us to easily create our games for that consoles is by using some tools for making our life a bit easier, such as sprite editors which didn't exist at that time or SDKs like the SGDK for the Mega Drive, which enables me to program the games using C and then compile it as a ROM.

Part 2

The jam had ended and the team needed another programmer to replace the other one, who didn't have more time to work on this project.

So here I appeared, a random person appeared from nowhere who started to solve some bugs and implement some new features, such as the Google Play's achievement system and a skin selector. Meanwhile the artists started to rework some assets as well as the musician and SFX artist polished some of their tracks.

After that we sent the compile to Google for publishing that on the Google Play Store, but one day later our game was suspended from there. We tried to hide de virus two more times and suddenly Google banned us from there because of the name, at least you can still play the game on itch.io

Conclusions

Problems

During the development of this project there were basically two problems:

  • My knowledge of Unity: At that time, my level of programming was quite low, basically being in that project helped me to learn a lot more of Unity and with that, be more prepared for future projects.
  • The programmer change problem: A huge problem was that the project was started by Balas and ended by me, that time when we changed was a bit confusing because you had a working game but with no idea how, so I lost some time trying to understand the whole logic of the project.

Improvements

After some time from this project, I can see that there are some improvements that could have been done.

  • Difficulty curve: The game in the beginning is really short but as you progress in it, each game becomes larger than the last one and then there appear situations where a single game is too long so the only way to end it is by dying and not by arriving to the end of the level.
  • Lack of content for engaging the player: We had skins and some achievements, but nothing more, at the beginning you started unlocking some but after that you started to lose interest in getting them and then got tired of playing the game, that's something that could have been fixed by adding an in-game currency with upgrades or maybe some powerups to unlock.
