Mega Drive Project

Making a Mega Drive game in 2021

The SEGA Mega Drive is quite an old console right? But that's no reason for not trying to make a game, so here I am, procrastinating while trying to accomplish my goal to complete a full game that can runs on original hardware without any problem.

The reason

Sometimes while I'm really bored I start thinking about weird projects to do and suddenly a random video pops up and for some reason that was about making a game for the NES.

When I went to search how expensive that was I remembered that I had a Mega Drive at home so after a quick Google search I found the solution with the name of SGDK.

After some time of doing nothing I got something, a ball moving across the screen, right after that I wanted more so I decided to work a bit on a project and then start a YouTube channel with the main goal of keeping me motivated for finishing that amazing project.

How

Nowadays, we make games using engines like Unity or Unreal, but there is still a few people that want to make games for "retro" consoles, such as the NES or in this case, the Mega Drive. This type of games receive the name of Homebrew Games, for being games made by hobbyists for proprietary hardware.

But what enables us to easily create our games for that consoles is by using some tools for making our life a bit easier, such as sprite editors which didn't exist at that time or SDKs like the SGDK for the Mega Drive, which enables me to program the games using C and then compile it as a ROM.

Despite of that, there are still some limitations such as the limited color palette of 512 colors only 64 of them that without the transparency end up being 60. The sound also has its limitations, you have to put a whole soundtrack inside a really small space, so what you make is using a tool such as Deflemask and write the songs there, then they are exported as .vgm files and the Mega Drive reads that as music sheets for its channels.

Hello, World!

Yes, if you write those lines of code and compile it using the SGDK you'll have a fully working "game" that can run on real hardware.

Now you might be wondering what do those lines mean, so well, let me explain:

The first line is obiously including the SGDK library with the functions and all the things you need for doing that and then we have the main function, the int main(). Right after that you have an SGDK function which tell the game to draw Hello, World! at the tiles 5 and 8, each one of them is 8px so it will be at the position 40px and 64px. Right after that there's an infinite loop with the SYS_doVBlankProcess(); function, which tells the game to end the current frame and start the next one and finally the return 0; which is because C needs to return something on that function.

Development

Progress made

Until now, I have been working bit by bit and made some progress:

  • Basic collision system: One of the things that were more difficult for me was to make the "physics system" which works like a grid collision system but with some freedom in it.
  • Scroll system: Another thing I have done is to make some metroidvania style room system where you reach the edge of a room and the game automatically transitions from one room to another smoothly.

The next step

As you can see, there isn't much progress yet, but what I am currently building is a strong base for not having to refactorize everything when I make new things.

  • Enemies: The next step that there's to come is to make enemies for the game, the game is a sidescroller similar to Shovel Knight and the game needs some enemies, so this will be my next step.
  • Destroyable map: With destroyable map I don't mean a map where everything can be destroyed, I mean that there will be some tiles that the player will be able to destroy by attacking them or using a certain ability, we'll see it on the future.
